# 2026-07-07 工作记录

## 静态新闻热点聚合站(计划阶段,尚未开发)

### 锁定决策
- 托管:GitHub Pages 纯静态
- 更新:GitHub Actions 定时抓取,每天 2 次(北京时间 09:00/18:00 ≈ UTC 01:00/10:00),支持手动触发
- 前端:原生 HTML/CSS/JS,零构建
- 领域:固定 16 个(科技/人工智能/人文/社会/教育/娱乐/医疗/财经/体育/游戏/汽车/美食/旅游/军事/国际/文化),每领域取前 6–13 条
- 卡片:基础版(标题+链接+来源+领域标签+热度/排名)
- 数据源:原定天行 tianapi(需免费 key)

### 用户追问的两个澄清点
1. 无 key 替代方案:除 tianapi 外,可行且无 key 的有——DailyHotApi 类开源聚合接口(覆盖几十平台热榜)、HN Algolia / GitHub Trending 官方无 key API(仅科技/AI)、RSS/RSSHub(无 key 但只有时间序无热度排序)。关键点:抓取在 Action 服务端 Node 跑,CORS 不构成限制,任何 HTTP 可取的源都可用。
2. 框架 vs 数据:静态页框架(index.html+assets)提交一次固定;每次抓取只重写 data/news.json(覆盖式),前端运行时 fetch 该 JSON 渲染。并非每次生成整页。Pages 每次 commit 会重发整站,但内容层仅 news.json 变化。

### 已确定(后续轮次)
- 数据源最终:无 key 公开聚合 = DailyHotApi 类接口(80+平台热榜,开源可自托管)
- 重要: DailyHotApi 按"平台"聚合非按"话题领域",故 16 领域映射 = 专属平台直接映射 + 综合平台(微博/知乎/百度)关键词过滤 两类
- 数据存存储方式变更:**不再覆盖单文件**,改为每次抓取写一份**按时间命名的 JSON**(如 data/snapshots/2026-07-07T01-00-00Z.json),支持历史回溯
- 需配套 data/manifest.json 登记所有快照(前端靠 manifest 列出可选日期,因 Pages 无目录列举)
- 前端默认显示最新,提供日期选择器回看历史
